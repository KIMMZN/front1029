웹 개발 개요
웹 개발의 정의
웹 개발은 사용자가 웹 브라우저를 통해 접근하는 웹 애플리케이션을 설계하고 구축하는 과정을 말합니다.

웹 브라우저의 역할
웹 개발에서 웹 브라우저는 클라이언트로서 사용자의 요청(Request)을 서버에 전달하고, 서버의 응답(Response)을 받아 웹 페이지를 표시합니다.

HTTP 프로토콜
웹 개발은 주로 HTTP(HyperText Transfer Protocol) 또는 HTTPS를 사용해 클라이언트와 서버 간에 데이터를 주고받습니다.

HTTP는 요청과 응답(Request-Response) 구조를 통해 웹 브라우저와 서버 간의 통신을 담당하는 주요 프로토콜입니다.
HTTPS는 HTTP에 SSL/TLS를 추가하여 보안을 강화한 버전입니다.
TCP/IP 통신 방식
웹 개발에서 사용하는 통신 방식은 TCP/IP입니다.

TCP(Transmission Control Protocol): 데이터의 전송을 신뢰성 있게 관리하며, 패킷을 순서대로 재조립하고 손실된 패킷을 재전송합니다.
IP(Internet Protocol): 각 장치에 고유한 IP 주소를 부여하여 패킷을 올바른 목적지로 전송합니다.
클라이언트-서버 모델
웹 개발은 클라이언트-서버 모델을 기반으로 합니다.

클라이언트(Client): 웹 브라우저를 통해 사용자가 입력한 정보를 서버로 전송하고, 서버로부터 응답을 받아 사용자에게 표시합니다.
서버(Server): 클라이언트의 요청을 처리하고, 데이터베이스 또는 애플리케이션 로직을 통해 결과를 생성하여 클라이언트에 응답합니다.
웹 통신의 과정

사용자가 웹 브라우저에 URL을 입력하거나 특정 액션을 수행합니다.
웹 브라우저는 입력된 정보를 바탕으로 **HTTP 요청(Request)**을 생성하여 서버에 전송합니다.
서버는 요청을 받아 처리한 후, **HTTP 응답(Response)**을 웹 브라우저에 전달합니다.
웹 브라우저는 서버로부터 받은 응답을 해석하여 사용자에게 결과를 표시합니다.
